# 课程论文可复用工作流程

本文件总结本项目从课程材料读取、选题、文献检索、论文写作到 Word 排版与 GitHub 归档的完整流程。以后撰写类似课程结课论文、文献综述、毕业设计文献部分时，可按本流程复用。

适用场景：

- 文献检索与论文写作课程论文；
- 医学、护理、检验、药学等专业的文献综述型结课论文；
- 需要避免重复选题、保留检索记录、形成 Word 正式稿的课程作业；
- 需要把资料来源、进展记录和最终文件同步到 GitHub 的项目。

## 1. 项目初始化

### 1.1 必备文件

新项目开始前，先确认根目录至少有以下材料：

| 文件 | 用途 |
| --- | --- |
| 课程任务书 | 确认字数、格式、评分标准、提交要求 |
| 论文模板 | 确认封面、目录、摘要、正文、参考文献等格式 |
| 选题表或已选题名单 | 避免重复选题 |
| `AGENTS.md` | 项目规则，约束日志、来源、隐私和 Git 流程 |
| `PROJECT_PROGRESS.md` | 进展记录 |
| `SOURCES.md` | 来源、文献、网页、工具记录 |
| `README.md` | 项目说明 |
| `.gitignore` | 排除隐私表格、Office 临时文件、缓存文件 |

若缺少 `PROJECT_PROGRESS.md` 或 `SOURCES.md`，应先创建，再开始检索和写作。

### 1.2 隐私处理

课程项目经常包含姓名、学号、联系方式、班级等个人信息。公开仓库前必须：

1. 检查原始选题表、模板、任务书是否含个人信息；
2. 含个人信息的原始表格不要直接提交到公开仓库；
3. 若需要保留查重依据，可制作脱敏摘要，只保留题目名称；
4. 提交前检查 Word、Excel 临时文件和生成副本；
5. 最终 DOCX 公开前扫描“姓名、学号、班级、电话、邮箱、身份证”等关键词。

推荐 `.gitignore` 规则：

```gitignore
~$*.doc
~$*.docx
~$*.xls
~$*.xlsx
*.tmp
*.temp
*.bak

# 原始含姓名/学号的选题表按实际文件名添加
文献检索结课论文选题表.xlsx
```

## 2. 读取任务书与拟定题目

### 2.1 任务书解读

先从任务书提取：

- 课程名称和论文类型；
- 正文字数下限；
- 是否要求 PPT；
- 是否必须自主命题；
- 参考文献数量和类型要求；
- 评分标准；
- Word 模板格式；
- 提交截止时间和提交文件类型。

将结论写入 `PROJECT_PROGRESS.md`。

### 2.2 选题查重

读取选题表，整理已有题目关键词。查重时不要只看题目表面文字，应比较：

- 疾病或研究对象是否相同；
- 检测指标是否相同；
- 临床应用场景是否相同；
- 是否只是换了说法但核心方向一致。

建议输出 `TOPIC_DUPLICATE_CHECK.md`，记录：

- 拟定题目；
- 与选题表相近的题目；
- 是否重复；
- 不重复的理由；
- 最终题目。

### 2.3 高分选题标准

优先选择满足以下条件的题目：

- 切口具体，不写空泛大题；
- 有权威指南、系统综述、Meta 分析或高水平原始研究可支撑；
- 能体现本专业特色；
- 有国内中文研究可补充；
- 能讨论方法学、质量控制、临床应用和局限；
- 不与选题表重复。

本项目最终题目示例：

> 可溶性fms样酪氨酸激酶-1/胎盘生长因子比值在子痫前期早期预测中的临床检验应用进展

## 3. 建立高分任务规划

建议创建 `COURSE_PAPER_TASK_PLAN.md`，包含：

- 当前任务边界；
- 选题查重结论；
- 选题高分理由；
- 研究问题；
- 评分标准对照表；
- 数据库与检索关键词；
- 纳入和排除标准；
- 文献管理字段；
- 章节结构和建议字数；
- 图表计划；
- 时间安排；
- 最终检查清单。

高分规划要把评分标准拆成可执行动作。例如：

| 评分维度 | 对应动作 |
| --- | --- |
| 学术规范 | DOI/PMID 核验，正文编号与参考文献对应 |
| 内容深度 | 按机制、方法、证据、局限、质控分层 |
| 文献综述 | 至少 15 条核心参考文献，优先指南和高水平研究 |
| 写作表达 | 摘要、引言、正文、结论、图表完整 |
| 专业特色 | 强化检测平台、样本、质控、报告解释 |

## 4. 文献检索与来源记录

### 4.1 建立检索记录

创建 `LITERATURE_SEARCH_RECORD.md`，每次检索都记录：

```text
日期：
数据库或网站：
检索式或关键词：
筛选条件：
获得结果：
拟使用文献或资料：
用于论文位置：
备注：
```

英文医学主题优先使用 PubMed、指南官网、期刊官网；中文主题可使用 CNKI、万方、维普、SinoMed、中文期刊官网和学校可访问数据库。

### 4.2 检索策略

推荐三层检索：

1. 宽检索：建立候选文献池；
2. 系统综述/Meta 检索：确定总体证据；
3. 临床应用/实施检索：找到管理路径、质量控制和现实应用证据。

医学主题示例：

```text
("sFlt-1/PlGF ratio" OR "sFlt-1:PlGF ratio" OR "soluble fms-like tyrosine kinase-1" OR "placental growth factor") AND (preeclampsia OR pre-eclampsia)
```

### 4.3 更新 SOURCES.md

每新增一个来源，都要写入 `SOURCES.md`：

| 字段 | 内容 |
| --- | --- |
| 来源名称 | 文献、网页、数据库、工具或本地文件 |
| 链接或文件名 | URL、DOI、PMID、文件路径 |
| 访问日期 | 实际访问日期 |
| 用途 | 用于题目、引言、正文、图表、工具等 |
| 备注 | 是否核心、候选、待核、隐私注意 |

## 5. 证据矩阵与精读笔记

### 5.1 证据矩阵

创建 `EVIDENCE_MATRIX.md`，按证据等级分层：

- 指南与诊断指导；
- 英文核心研究与综述；
- 方法学和质量控制研究；
- 中文临床研究；
- 待核线索。

推荐字段：

| 状态 | PMID/链接 | 作者与年份 | 文献类型 | 关键用途 | 拟用于论文位置 |
| --- | --- | --- | --- | --- | --- |

状态可用：

- `核心`：大概率进入最终参考文献；
- `候选`：可补充背景、方法或局限；
- `待核`：题目相关但未完全核实。

### 5.2 精读笔记

创建 `CORE_LITERATURE_NOTES.md`，只记录已核验信息：

- DOI 或 PMID；
- 研究设计；
- 样本量；
- 对象和孕周/疾病阶段；
- 检测平台或方法；
- 截断值；
- 主要结果；
- 局限性；
- 可写入论文的论点。

原则：不能把未全文核实或只看搜索结果的信息写成确定结论。

## 6. 论文大纲与正文写作

### 6.1 详细大纲

创建 `COURSE_PAPER_DETAILED_OUTLINE.md`，包括：

- 写作定位；
- 核心论点；
- 章节和建议字数；
- 段落级写作提纲；
- 图表计划；
- 写作风险控制。

文献综述型课程论文建议结构：

1. 摘要与关键词；
2. Abstract 与 Keywords；
3. 引言；
4. 文献检索策略与筛选过程；
5. 机制或理论基础；
6. 检测方法或研究方法；
7. 核心证据综述；
8. 临床应用、局限与质量控制；
9. 文献检索与写作反思；
10. 结论；
11. 参考文献。

### 6.2 正文写作顺序

推荐顺序：

1. 先写检索策略、机制基础、方法学；
2. 再写核心证据章节；
3. 再写局限、质量控制、未来方向；
4. 最后写摘要、结论和反思；
5. 统一参考文献编号。

写作时每个关键论点都要能回到 `EVIDENCE_MATRIX.md` 或 `CORE_LITERATURE_NOTES.md`。

### 6.3 避免过度结论

医学综述尤其要避免：

- 把预测价值写成确诊能力；
- 把 AUC 高写成临床结局一定改善；
- 忽略人群、平台、孕周、截断值差异；
- 把待核中文线索当成强证据；
- 只写优点，不写局限和质量控制。

## 7. 图表设计

推荐至少包含：

| 图表 | 作用 |
| --- | --- |
| 文献检索与筛选流程表 | 展示课程“文献检索”能力 |
| 核心证据矩阵 | 展示综述证据分层 |
| 指标或方法比较表 | 展示专业分析能力 |
| 临床应用路径图 | 展示从证据到实践的转化 |

Word 转换时，宽表容易不可读。建议：

- Markdown 中可保留简表；
- DOCX 生成时用脚本插入真实 Word 表格；
- 表格控制在 2-3 列；
- 长句拆短，避免单元格挤压；
- 表头加灰底，正文 9-10.5 磅；
- 渲染成 PDF/PNG 后检查表格是否跨页、截断或文字重叠。

## 8. Word 正式版生成

本项目使用的稳定方案：

1. Markdown 写正文；
2. Pandoc 读取课程模板生成 DOCX；
3. Python 脚本用 `python-docx` 修正封面、目录、表格、页眉页脚等；
4. LibreOffice 导出 PDF；
5. Poppler 渲染 PNG；
6. 人工查看关键页面。

本项目已有脚本：

```powershell
python .\scripts\build_course_paper_docx.py
```

如果迁移到新项目，可复制并修改该脚本，使其适配新论文题目、模板和表格。

## 9. 渲染检查

Word 文件不能只看 DOCX 是否生成成功，必须渲染检查：

```powershell
& 'C:\Program Files\LibreOffice\program\soffice.com' --headless --norestore --convert-to pdf --outdir $env:TEMP .\COURSE_PAPER_FORMATTED.docx
pdfinfo <pdf-path>
pdftoppm -png -r 120 <pdf-path> <output-prefix>
```

重点检查：

- 封面是否有个人信息占位而非真实隐私；
- 目录页码是否合理；
- 摘要和关键词是否同页或自然分页；
- 表格是否有边框、表头、重复表头和合适列宽；
- 图是否清晰；
- 章节标题是否悬空；
- 参考文献是否完整；
- 是否存在空白页、乱码、裁切、重叠。

## 10. 进展记录与 GitHub 推送

每个有意义阶段都要追加 `PROJECT_PROGRESS.md`：

```text
## 日期 工作标题

- 操作者：
- 工作目标：
- 修改或新增文件：
- 使用来源：
- 工作内容：
- 验证结果：
- git 状态：
```

推荐 Git 命令：

```powershell
git -c core.longpaths=true status --short
git -c core.longpaths=true add <files>
git -c core.longpaths=true commit -m "docs: describe this step"
git -c core.longpaths=true push origin main
```

提交前必须确认：

- 没有提交原始含姓名/学号的选题表；
- 没有提交 Office 临时文件；
- 没有提交未经确认公开的个人信息；
- `PROJECT_PROGRESS.md` 和 `SOURCES.md` 已同步；
- `git diff --check` 通过。

## 11. 最终交付检查清单

- [ ] 题目不与选题表重复；
- [ ] 正文超过任务书字数下限；
- [ ] 摘要、关键词、Abstract、Keywords 完整；
- [ ] 文献检索策略写清楚；
- [ ] 至少 15 条可核验参考文献；
- [ ] 每条核心证据有 DOI、PMID、指南链接或期刊页面；
- [ ] 正文引用编号与参考文献对应；
- [ ] 有检索流程、证据矩阵或专业比较表；
- [ ] 有局限性、质量控制、学术伦理或写作反思；
- [ ] Word 模板格式基本符合课程要求；
- [ ] PDF/PNG 渲染检查通过；
- [ ] 隐私扫描通过；
- [ ] `PROJECT_PROGRESS.md`、`SOURCES.md` 完整；
- [ ] GitHub 已提交并推送。

## 12. 可复用提示词

在新课程论文项目中，可直接使用下面提示词：

```text
请按课程论文复用流程工作：
1. 先阅读 AGENTS.md、PROJECT_PROGRESS.md、SOURCES.md、任务书、论文模板和选题表；
2. 检查隐私与选题重复；
3. 拟定一个不重复且有高水平文献支撑的题目；
4. 建立任务规划、检索记录、来源记录、证据矩阵和精读笔记；
5. 撰写 Markdown 初稿，包含摘要、引言、检索策略、正文、反思、结论和参考文献；
6. 生成 Word 正式版，并用 LibreOffice/PDF/PNG 做视觉检查；
7. 每次修改都更新 PROJECT_PROGRESS.md 和 SOURCES.md；
8. 每个有意义阶段提交并推送 GitHub；
9. 不提交含姓名、学号、联系方式等隐私的原始文件。
```
